You are running the `/fingerprint` slash command from the `bug-bounty-recon` plugin.

The user's target is: **$ARGUMENTS**

If `$ARGUMENTS` is empty or contains whitespace/special characters that are not valid in a domain name, stop and print:

> Usage: /fingerprint <target-domain>
> Example: /fingerprint example.com

Otherwise proceed with the steps below **in order**. Do NOT skip preflight. Do NOT ask the user for confirmation between steps.

---

## Step 1 — Tool preflight

Run this single bash command and capture output:

```bash
for t in subfinder amass assetfinder findomain chaos dnsx httpx katana gospider hakrawler waybackurls gau subjs linkfinder nmap rustscan jq; do
  if ! command -v "$t" >/dev/null 2>&1; then echo "MISSING: $t"; fi
done
```

If any `MISSING:` lines appear, print them and stop with:

> Required tools are missing. Install them in your Kali VM and re-run /fingerprint.

Do NOT proceed.

## Step 2 — API key preflight

Run:

```bash
[ -n "$PDCP_API_KEY" ] && echo "PDCP_API_KEY: set" || echo "PDCP_API_KEY: MISSING"
```

If `MISSING`, print:

> The chaos tool requires a ProjectDiscovery API key.
> Set it in your shell and re-run:
>     export PDCP_API_KEY="your-key-here"
> Get a key at https://cloud.projectdiscovery.io

Stop. Do NOT proceed.

## Step 3 — Scope file

If `./scope.txt` does not exist in the current working directory, create it:

```bash
if [ ! -f ./scope.txt ]; then
  cat > ./scope.txt <<EOF
# Auto-generated by /fingerprint on $(date -Iseconds)
# Edit this file to restrict scope before re-running.
*.$ARGUMENTS
EOF
fi
```

Print the resulting scope file contents so the user can confirm.

## Step 4 — Workspace setup

```bash
TARGET="$ARGUMENTS"
OUTPUT_DIR="./recon/$TARGET"
mkdir -p "$OUTPUT_DIR/subdomains_by_status" \
         "$OUTPUT_DIR/urls_by_subdomain" \
         "$OUTPUT_DIR/portscan" \
         "$OUTPUT_DIR/subdomains"
```

## Step 5 — Dispatch the orchestrator

Use the Agent tool with `subagent_type: recon-orchestrator` and this prompt:

> You are the recon orchestrator for target `$ARGUMENTS`.
>
> Environment:
> - TARGET=$ARGUMENTS
> - OUTPUT_DIR=./recon/$ARGUMENTS
> - SCOPE_FILE=./scope.txt
> - RATE_LIMIT=10
> - TIMEOUT=300
>
> Drive the full 12-agent chain as defined in your system prompt. Begin with agent-1.

When the orchestrator returns, print its summary verbatim to the user.
